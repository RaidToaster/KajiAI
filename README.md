# KajiSF-DJ

**Kaji** (Indonesian/Malay: "to investigate / examine") + **SF** (SourceForce) + **DJ** (Disinformation Jammer)

AI-assisted evidence aggregation and claim analysis platform. Helps users investigate factual claims, surface contradictions, evaluate source quality, and organise evidence transparently.

> This analysis was generated by an AI system and may contain errors. Always verify claims using primary sources and independent research.

## Architecture

```
Claim → Intake → Decomposition → Parallel Verification → Verdict → Report
```

Profile-driven CrewAI architecture: one BaseCrew, many domain profiles (scientific, political, health, general).

## Project Structure

```
src/kaji/
├── config/          # YAML: agents, tasks, profiles, policies, models, verdict rules
├── crews/           # BaseCrew, profile loader, execution context
├── routing/         # Classifier, decomposer, tool selector, domain router
├── execution/       # Policy engine, budget manager, retry controller
├── agents/          # Agent role implementations
├── tools/           # Search, scraping, RAG tool integrations
├── verification/    # Evidence validator, citation checker, contradiction scanner
├── verdict/         # Engine, labels, confidence, ranking, consensus
├── cache/           # Evidence, search, scrape caches
├── output/          # Formatter, templates, disclaimer
├── interfaces/      # CLI, API (FastAPI), Gradio UI
└── models/          # LLM provider abstraction
```

## Principles

- **Epistemic Honesty** — never imply certainty beyond available evidence
- **Evidence Before Intelligence** — agents reason only over retrieved evidence
- **Modularity** — all subsystems independently configurable
- **Transparency** — expose contradictions, gaps, and uncertainty
- **Governance** — agents constrained by policy, not fully autonomous
- **Research Assistance** — humans make final judgment

## Phases

1. **MVP** — Core fact-checking with Tavily, Serper, Firecrawl
2. **Advanced Governance** — Caching, budgeting, contradiction analytics
3. **Multilingual** — Indonesian, English support
4. **RAG** — Curated fact-check archives, local knowledge bases
5. **Collaborative** — User annotations, human review mode
